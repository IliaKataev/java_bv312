# Техническое задание на проект "Город из кубиков"

## 1. Функциональные задачи

### 1.1. Динамические типы зданий
- Убрать жесткую привязку к конкретным классам (`House`, `School`, `Police`).
- Добавить возможность создавать новые типы зданий.
- Каждое здание должно иметь:
    - Название
    - Количество требуемых кубиков

### 1.2. Хранение состояния города в JSON
- Сохранение в БД.
- Проверить все сценарии:
    - Добавление зданий
    - Добавление кубиков
    - Сброс города
- После перезагрузки или перезапуска приложения данные не должны теряться.

### 1.3. Очистка города
- Добавить функциональность сброса города до дефолтного состояния:
    - Пустой список построек
    - Начальный запас кубиков
- Состояние должно сохраняться в JSON.

### 1.4. Сортировка построек
- Добавить возможность сортировки построек:
    - По типу
    - По дате постройки
- В интерфейсе реализовать выбор критерия сортировки.

### 1.5. История построек
- Вести историю всех построек:
    - Тип здания
    - Дата и время постройки
- Сделать отдельную страницу или панель для просмотра истории, чтобы не загромождать главную страницу.

### 1.6. Иконки вместо названий
- Вместо текстового отображения зданий на главной странице использовать иконки.
- Иконка должна соответствовать типу здания.

### 1.7. Изменяемое ограничение на количество зданий
- Добавить возможность изменять максимальное количество построек через выполнение миссий (см. пункт 2.2).

---

## 2. Креативные задачи

### 2.1. События в городе
- Реализовать систему событий, которые происходят через определенный интервал времени.
- Пример события: пожар
    - Каждые 3 секунды уменьшает запас кубиков на случайное количество (1–5)
    - Уничтожает одно здание каждые 15 секунд
    - Длительность события варьируется от 16 секунд до 3 минут
- События должны отображаться в интерфейсе (например, лог или уведомление).

### 2.2. Цели и миссии
- Реализовать случайные задания/миссии для игрока.
- Пример миссии: построить 5 домов и 1 школу
- При выполнении миссий учитывается:
    - Текущее количество построек
    - Максимальный лимит зданий
- Бонусы за выполнение миссий:
    - Увеличение максимального числа зданий
    - Дополнительный запас кубиков

---

## 3. Работа с базой данных

### 3.1. Перенос хранения данных в БД
- Использовать базу данных для хранения информации о:
    - Состоянии города
    - Построенных зданиях (здание имеет свойства количество-кубиков, название)
    - Запасе кубиков
    - Истории действий
    - Выполненных миссиях
- Обеспечить корректную загрузку состояния после перезапуска приложения.
